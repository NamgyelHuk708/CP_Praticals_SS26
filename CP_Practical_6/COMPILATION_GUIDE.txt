# Compilation and Execution Guide

## Overview
This guide provides step-by-step instructions to compile and run the three algorithm implementations.

## System Requirements
- C++ Compiler (g++ or clang++)
- Linux/Unix environment or Windows with WSL
- Terminal/Command Prompt access

## File Structure
```
CP_Practical_6/
├── trie.cpp              # Trie implementation
├── patricia.cpp          # PATRICIA tree implementation  
├── manacher.cpp          # Manacher's algorithm
├── README.md             # Project summary
├── REFLECTION.md         # Detailed reflection
├── OUTPUT_SCREENSHOTS.txt # All program outputs
└── COMPILATION_GUIDE.txt # This file
```

## Compilation Instructions

### Option 1: Individual Compilation
```bash
# Compile Trie
g++ -o trie trie.cpp

# Compile PATRICIA
g++ -o patricia patricia.cpp

# Compile Manacher
g++ -o manacher manacher.cpp
```

### Option 2: Batch Compilation (Linux/Mac)
```bash
g++ -o trie trie.cpp && g++ -o patricia patricia.cpp && g++ -o manacher manacher.cpp
```

### Option 3: Using Makefile (if available)
```bash
make          # Compiles all programs
make clean    # Removes executables
```

## Execution Instructions

### Run Trie Algorithm
```bash
./trie
```
Expected output: Demonstrates insert, search, delete operations on words

### Run PATRICIA Tree
```bash
./patricia
```
Expected output: Shows compressed trie operations with various keys

### Run Manacher's Algorithm
```bash
./manacher
```
Expected output: Multiple test cases showing palindrome detection

### Run All Programs Sequentially
```bash
./trie && ./patricia && ./manacher
```

### Redirect Output to File
```bash
./trie > trie_output.txt
./patricia > patricia_output.txt
./manacher > manacher_output.txt
```

## Troubleshooting

### Issue: Command not found
**Solution**: Ensure you're in the correct directory
```bash
cd /home/namgyel/Desktop/SEM_6/CSF303/CP_Praticals_SS26/CP_Practical_6
```

### Issue: Compilation error about includes
**Solution**: The programs use only standard C++ library, ensure:
- iostream is available
- string is available
- vector is available
```cpp
#include <iostream>
#include <string>
#include <vector>
```

### Issue: Program runs but no output appears
**Solution**: Output is sent to stdout, verify terminal settings

### Issue: Segmentation fault
**Solution**: Check for memory leaks in test data or stack overflow

## Verification Steps

1. **Check compilation success**: Look for executable files
   ```bash
   ls -la trie patricia manacher
   ```

2. **Run each program**: Verify output appears
   ```bash
   ./trie | head -20
   ```

3. **Check exit codes**: Should be 0 on success
   ```bash
   ./trie; echo $?
   ```

## Customization

### To modify test cases:
1. Edit the main() function in the respective .cpp file
2. Change the test data in the vector/array declarations
3. Recompile using g++

### Example - Adding more test words to Trie:
```cpp
// Original
trie.insert("hello");
trie.insert("world");

// Modified
trie.insert("hello");
trie.insert("world");
trie.insert("yourword");  // Add new word
```

## Support Information

For any compilation or execution issues:
1. Check C++ compiler is installed: `g++ --version`
2. Ensure file permissions: `chmod +x *.cpp`
3. Verify file encoding: Files should be UTF-8 ASCII
4. Check for typos in filenames (case-sensitive on Linux)
